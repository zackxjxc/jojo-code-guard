# 生效与验收

本文档说明规则在哪个运行阶段生效，以及每一层能保证什么。它是安装和验收说明，不是要求用户每天输入命令的工作流。

## 触发矩阵

| 场景 | 入口 | 触发时机 | 自动程度 | 责任边界 |
| --- | --- | --- | --- | --- |
| 会话开始 | Claude 插件 `SessionStart` | 启动、恢复、清空或压缩会话 | 已安装并启用插件时自动 | 注入主 Skill 和项目 `AGENTS.md`；插件未加载时不能假称已注入 |
| 会话开始 | Codex 主 Skill | Skill Discovery 选择 Skill 时 | 由客户端/模型按描述选择；不是系统级 Hook 保证 | 允许隐式调用；需强制时，将规则合并到用户级 `AGENTS.md` |
| 会话开始 | Codex `SessionStart` | 当前实测版本发现且信任 `hooks/hooks.json` 后 | 需客户端支持并信任 Hook | 普通 stdout 注入主 Skill 和项目 `AGENTS.md`；未信任时只可发现，不会执行 |
| AI 写文件后 | Claude `PostToolUse` | 各类文件写入工具完成后 | 已安装并启用插件时自动 | 运行 `post-write-check`；写入已发生，不能回滚 |
| AI 写文件后 | Codex | 主 Skill 修改闭环；当前实测版本从 `hooks/hooks.json` 发现 Hook | 需客户端支持并信任 Hook | Hook 从业务仓库 cwd 启动，通过客户端注入的插件根路径找脚本；不可用时由 AI 在每次写入后调用 `check_diff.py` |
| 用户主动检查 | `doctor`、`check-diff`、`help` | 用户明确要求时 | 手动低频入口 | 只读检查或展示说明；修复、同步和安装前要求确认 |
| Git 提交阶段 | 仓库私有 `pre-commit` | 用户初始化并安装后 | 机械自动 | 只检查暂存区，不替代 AI 的最小改动规则 |

生命周期 Hook 找不到主 Skill 时，会向模型注入明确的加载失败与暂停要求；这不能替代完整规则，只用于避免资源损坏时静默继续写文件。

## 结论

脚本不会在模型每个思考步骤或每个输出 token 上运行。会话脚本只在会话生命周期事件触发，写入脚本只在客户端报告文件工具完成后触发，`pre-commit` 只在 Git 提交阶段触发；`doctor` 和 `check-diff` 是用户要求时的低频入口。其余保护要求由主 Skill 和这些 Markdown 规则指导 AI 执行；Hook 不可用时，AI 必须在每次写入后自动调用 `check_diff.py`。

## 规则分层

- 自然语言规则位于主 `skills/jojo-code-guard/SKILL.md`、本目录的 `全局规则.md` 和本文件；
  它们指导 AI 在编辑前后读取项目规则、保留原始字节并审阅差异。
- 确定性规则位于 `scripts/guard_core.py`、`check_diff.py`、`hook_check.py` 以及插件 Hook 脚本；
  它们检查编码、BOM、换行、末尾换行、替换字符、Git 空白、异常 diff、权限位和文件类型变化。
- `.editorconfig` 与 `.gitattributes` 只提供保守的项目提示和 Git 字节策略。老文件不强制统一编码或换行；新增文件由检查脚本验收。
- `CHANGELOG.md` 中旧版本对提交 Skill 的记载是历史说明，不是当前入口；当前没有 `commit` Skill 或 `commands/commit.md`。

## 一次初始化

1. 安装插件并重新打开会话；若客户端要求信任 Hook，首次安装或 Hook 内容变化的升级后审阅并确认，再确认实际加载了新版本 Skill。
2. 让 `doctor` 先输出只读报告；审阅影响后，再按需使用 `--repair` 补齐仓库保守配置。
3. 需要 Git 提交阶段机械门禁时，再明确确认并安装仓库私有 `pre-commit`；不需要时可以不安装。
4. 希望 Codex 在每个会话都收到强制规则时，预览后将受管规则合并到用户级 `~/.codex/AGENTS.md`；该同步只做一次，之后日常写代码不需要输入守护命令。

## 验收标准

- Claude 新会话能看到主 Skill；一次文件写入后能看到 `PostToolUse` 检查结果。
- Codex 的插件清单能通过当前 validator，主 Skill 能在 `debug prompt-input` 的可用 Skill 列表中发现；当前实测的 Codex 0.142.3 能从 `hooks/hooks.json` 发现 `SessionStart` 和 `PostToolUse`，且命令从业务仓库 cwd 执行时仍能通过客户端注入的插件根路径找到脚本。生命周期 Hook 是否真正运行还必须按目标版本、功能开关和信任状态验证。
- 故意把已有文件整体改成另一种换行、编码或 BOM 时，`check_diff.py` 返回阻断诊断。
- 故意在新增工具文件中使用错误编码、BOM、换行或缺少末尾换行时，检查返回阻断诊断。
- 安装了 `pre-commit` 的仓库在暂存污染时提交被拦截；未安装时，其他日常守护仍可正常使用。
- 工作区没有因检查而被自动格式化、转码或改变 Git 状态。
